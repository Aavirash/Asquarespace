## 2. Goal

Ship a smooth iPad app replica of the existing canvas UX (including touch + Apple Pencil), while preserving current visual behavior and interactions.

## 3. Recommended Architecture

- Wrapper: Capacitor (`@capacitor/core`, `@capacitor/ios`, `@capacitor/cli`)
- UI runtime: Existing HTML/CSS/Vanilla JS (no framework migration required)
- Native host: iOS WebView app generated by Capacitor
- Persistence on iPad: browser storage first; optional migration to Capacitor Filesystem later

## 4. Pre-Build Checklist (When Xcode Finishes)

1. Open iOS project:
   - `npx cap open ios`
2. In Xcode:
   - Select a Team under Signing & Capabilities
   - Set Bundle Identifier (`com.asquarespace.canvas` or your preferred one)
   - Select your iPad as run target
   - Build/Run once
3. If signing errors appear:
   - Ensure Apple ID is added in Xcode settings
   - Use a unique bundle id

## 5. Web Asset Workflow

Because Capacitor serves files from `www`, use this refresh loop after web changes:

1. Copy web assets into `www`
2. Run `npx cap sync ios`
3. Rebuild from Xcode

## 6. Code Work Required for iPad Quality

### 6.1 Input Model Upgrade (High Priority)

Current desktop flow relies heavily on mouse events.

Planned change:
- Move interaction handling to Pointer Events:
  - `pointerdown`, `pointermove`, `pointerup`, `pointercancel`
- Keep mouse behavior intact while adding touch/pencil support.
- Handle multi-touch gestures safely:
  - one finger: drag/select
  - two fingers: pan/zoom
- Add `touch-action` CSS where needed to prevent gesture conflicts.

Acceptance criteria:
- Node drag/scale/rotate works with finger and Pencil
- Canvas pan/zoom feels stable at 60fps+
- No accidental text selection during gestures

### 6.2 Gesture Conflict Resolution (High Priority)

Need deterministic gesture priority:
1. Active transform handle gesture wins
2. Node drag gesture next
3. Canvas pan/zoom fallback

Acceptance criteria:
- No stuck drag states
- No jumpy resize/rotate on touch start

### 6.3 Performance Tuning (Medium Priority)

- Keep transforms on GPU-friendly properties (`transform`, `opacity`)
- Avoid expensive layout reads in move loops
- Throttle expensive observers if needed

Acceptance criteria:
- Smooth interactions on iPad without visible frame drops

### 6.4 Native Integrations (Optional, Later)

- Share sheet export
- Files app import/export
- Haptics on key interactions

## 7. Feature Compatibility Notes

### 7.1 Adobe / CEP-specific functionality

The AE extension bridge (`CSInterface`, JSX calls) will not work on iPad.

Plan:
- Hide or disable AE-specific buttons in iPad build profile
- Preserve canvas generation, notes, board management, local media browsing

### 7.2 Local paths and desktop assumptions

Any desktop-only filesystem assumptions should be guarded behind platform checks.

## 8. Build Profiles Strategy

Use one codebase, two profiles:
- Desktop CEP profile (current)
- iPad Capacitor profile

Implementation suggestion:
- Add a runtime platform flag and conditionally render unsupported controls.
